# floc-reader 计划文档（V1）

## 1. 项目目标

打造一款**本地化、无广告、可离线、阅读体验可高度自定义**的小说阅读 App。  
核心是「打开即阅读」，减少干扰，优先满足沉浸式看书需求。

---

## 2. 问题与价值

### 当前痛点

- 广告多、会员门槛高、阅读成本高。
- 阅读界面信息过载，无法专注。
- 常见 App 的亮度、行距、字体、背景、翻页效果不够灵活。
- 依赖网络，离线场景（如飞机）无法阅读。

### 产品价值

- 本地缓存章节，弱网/无网可读。
- 极简 UI，书架即首页，减少学习成本。
- 可配置阅读体验，适配不同用户习惯。
- 阅读进度和书签自动保存，任何退出场景都不丢失。

---

## 3. 用户与场景

### 目标用户

- 频繁阅读网文/小说的用户。
- 对广告敏感、希望低成本阅读的用户。
- 在通勤、飞机等离线场景有阅读需求的用户。

### 关键场景

- 首次进入：看到简洁书架，快速开始阅读。
- 选择一本书：进入章节列表，有历史记录则直达上次阅读页。
- 阅读中：通过手势翻页、长按调设置，不中断阅读流。
- 异常退出：再次打开仍可从原进度继续。

---

## 4. 产品范围（MVP / 后续）

## 4.1 MVP（必须做）

- 书架首页（简洁明亮）
  - 书籍卡片列表（封面、书名、作者、阅读进度）。
  - 支持按最近阅读排序。
- 分类能力（基础）
  - 自定义分类（如：在读 / 想读 / 完结 / 收藏）。
- 书籍详情与章节列表
  - 展示章节目录。
  - 存在历史记录时，按钮文案显示「继续阅读」并直达对应页。
- 阅读器
  - 翻页手势（点击左右分区 + 滑动翻页）。
  - 长按弹出 BottomSheet 设置。
  - 设置项：亮度、行距、字体大小、夜间模式。
- 本地持久化
  - 章节缓存到本地。
  - 阅读进度/书签在任何退出场景都自动保存。

## 4.2 V1.5（建议追加）

- 书架高级交互（拖拽排序、分组折叠、最近阅读浮层入口）。
- 阅读器主题（护眼色、纸张纹理、更多字体）。
- 翻页动画可选（仿真、覆盖、无动画）。

## 4.3 V2（探索）

- 多书源策略（源管理、可用性评分、失败自动切换）。
- 书源抓取规则可配置化（便于维护）。
- 阅读统计（阅读时长、完成率）。

## 5. 关键交互设计（重点）

## 5.1 书架与阅读流程

1. 启动 App -> 书架首页。
2. 点击书籍 -> 章节页。
3. 如果存在历史记录 -> 直接进入历史页码（并可从目录跳转）。
4. 阅读时长按 -> BottomSheet 设置。

## 5.2 阅读器手势规则

### 翻页规则

- 屏幕水平二分：
  - 单击右半屏：下一页。
  - 单击左半屏：上一页。
- 滑动规则：
  - 左滑：下一页。
  - 右滑：上一页。

### 与系统返回手势兼容（安卓重点）

- 避免与系统边缘返回冲突：
  - 在屏幕左右边缘各预留 `edgeSafeZone`（建议 24dp~32dp）不拦截横向手势。
  - 仅在中间可交互区域识别阅读翻页滑动。
- 点击手势可全屏生效，滑动手势应避开边缘返回区域。
- 从阅读页返回 -> 回到书架（或上一层：章节页，按最终导航结构确定）。

## 5.3 设置 BottomSheet

- 触发：阅读页长按。
- 内容：
  - 亮度滑杆（优先 App 内亮度覆盖，无法覆盖时回退系统亮度）。
  - 行距滑杆。
  - 字体大小滑杆。
  - 夜间模式开关。
- 生效策略：实时预览 + 立即持久化。

## 6. 数据与持久化设计

## 6.1 关键数据模型（建议）

- `Book`：id、title、author、cover、categoryId、sourceId、lastReadChapterId、lastReadOffset。
- `Chapter`：id、bookId、title、index、content、updatedAt、cached。
- `ReadingConfig`：fontSize、lineHeight、themeMode、brightness。
- `Bookmark`：bookId、chapterId、offset、updatedAt。
- `Source`：id、name、baseUrl、enabled、priority。

## 6.2 存储建议

- 本地数据库：SQLite（或同类轻量方案）存结构化数据。
- 章节正文：数据库或文件缓存二选一（MVP 建议先放数据库，开发快）。
- 配置项：键值存储（如 SharedPreferences）。

## 6.3 退出即保存（强约束）

- 在以下时机统一触发持久化：
  - 页码变化时节流保存（如每 1~2 秒或翻页后）。
  - App 生命周期 `inactive/paused/detached`。
  - 路由离开阅读页前。
- 目标：任何崩溃、杀进程、手动退出后，阅读进度回放误差 <= 1 页。

## 7. 书源与抓取方案（风险最高）

> 你提到《最后一个道士》（夏忆）免费接口暂未找到，这里按「可替换书源层」设计，先保证产品可用，再迭代资源接入。

### 7.1 MVP 策略

- 第一阶段先支持**手动导入本地文本/epub/txt**（保障可读可测）。
- 第二阶段接入单一可用网络书源（接口/爬取）。
- 第三阶段扩展多书源并做源切换。

### 7.2 技术建议

- 抽象 `SourceAdapter` 接口：
  - `search(keyword)`
  - `getBookDetail(bookId)`
  - `getChapters(bookId)`
  - `getChapterContent(chapterId)`
- 各书源独立适配，避免互相耦合。
- 增加缓存与重试机制，降低接口波动影响。

### 7.3 合规提醒

- 仅接入有授权或明确可使用的公开资源。
- 避免分发侵权内容，保留源站信息与必要声明。

## 8. UI/交互参考方向（书架不做“普通列表”）

### 推荐参考类型

- 卡片式书架 + 分组（在读/最近更新/收藏）。
- 「最近阅读」置顶横滑区（提升续读效率）。
- 支持编辑态：批量移动分类、批量缓存、批量删除。
- 空状态友好引导（导入本地书籍 / 添加书源）。

### 设计原则

- 简洁明亮：弱化装饰，突出书籍与阅读入口。
- 首屏可用：打开后 1 秒内看到可点击书籍。
- 操作可预期：所有核心动作都在 1~2 步可达。

## 9. 技术架构（Flutter）

- 技术栈结论：本项目统一使用 Flutter。
- 选型原因：跨端一致性好、手势体系完善、文本排版与渲染能力强，适合阅读器场景。
- 落地原则：优先采用 Flutter 生态成熟方案，必要时通过平台通道补充原生能力。

### 分层

- `presentation`：页面与组件（书架/章节/阅读器/设置面板）。
- `domain`：阅读规则、翻页逻辑、进度保存策略。
- `data`：本地库、缓存、书源适配器、网络请求。

### 关键模块

- 阅读内核（分页/排版/进度计算）。
- 手势控制器（翻页与边缘返回冲突处理）。
- 书源引擎（可插拔）。
- 同步与缓存管理器（章节下载、过期更新）。

## 10. Flutter 依赖与工程建议清单

> 版本原则：优先使用稳定版最新依赖，定期统一升级，避免长期锁死旧版本。

### 10.1 推荐依赖（按模块）

- 状态管理：`flutter_riverpod`
  - 理由：依赖注入清晰、可测试性好，适合中长期演进。
- 路由管理：`go_router`
  - 理由：声明式路由，适合书架 -> 章节 -> 阅读器的层级导航。
- 网络请求：`dio`
  - 理由：拦截器、重试、超时配置完善，便于书源层统一处理。
- 本地数据库：`drift` + `sqlite3_flutter_libs`
  - 理由：类型安全、查询能力强，适合书籍/章节/进度等结构化数据。
- 轻量配置存储：`shared_preferences`
  - 理由：适合字体、行距、主题、亮度等阅读偏好设置。
- JSON 模型：`json_serializable` + `build_runner`
  - 理由：减少手写序列化错误，提升维护效率。
- 不可变模型（可选）：`freezed`
  - 理由：状态对象清晰，减少可变状态导致的 bug。
- 屏幕亮度：`screen_brightness`
  - 理由：支持 App 内亮度调节，满足阅读页亮度需求。
- 本地文件访问：`file_picker` + `path_provider`
  - 理由：支持导入本地 txt/epub 与缓存目录管理。
- 日志与崩溃：`logger`（基础）+ `sentry_flutter`（可选）
  - 理由：便于定位分页/手势/书源异常问题。

### 10.2 MVP 阶段可暂缓依赖

- 动画增强：`animations`、复杂自定义翻页引擎。
- 图片缓存增强：`cached_network_image`（封面量大时再引入）。
- 国际化：`intl`（先做中文单语版可延后）。

### 10.3 最小工程目录建议

- `lib/app/`：应用入口、主题、路由、全局配置。
- `lib/features/bookshelf/`：书架与分类。
- `lib/features/catalog/`：章节目录。
- `lib/features/reader/`：阅读器、手势、设置面板。
- `lib/features/source/`：书源接口与适配器。
- `lib/data/`：数据库、DAO、本地缓存、网络客户端。
- `lib/domain/`：实体、用例、仓库抽象。
- `lib/shared/`：通用组件、工具类、常量。

### 10.4 工程约束（保证可维护）

- 统一 `dart format` 与 `flutter_lints`，提交前跑静态检查。
- 页面状态只通过状态管理更新，避免 UI 层直接改数据库。
- 书源请求统一走 `dio` 客户端，禁止页面直接发请求。
- 阅读进度保存使用统一服务，避免多处重复实现。

## 11. MVP 验收标准

- 无网环境下可正常阅读已缓存章节。
- 阅读页翻页手势与安卓返回手势不冲突。
- 长按可打开设置并实时生效。
- 任何退出方式后都能恢复到上次阅读位置（误差 <= 1 页）。
- 启动即见书架，核心路径（打开书 -> 阅读）不超过 2 次点击。

---

## 12. 风险与应对

- 书源不稳定：做适配器隔离 + 缓存 + 多源预案。
- 排版性能：长文本分页预计算 + 惰性加载。
- 手势冲突：边缘安全区 + 真机专项测试（不同品牌安卓）。
- 数据丢失：多时机保存 + 崩溃恢复验证。
